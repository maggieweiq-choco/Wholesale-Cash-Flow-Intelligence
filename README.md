# Wholesale Cash Flow Intelligence

> **H0 Hackathon · B2B Track · Jun 29 2026**
>
> Tells wholesale business owners: "On day 45, you'll be short $23,000 — should you liquidate inventory or take a loan?"

---

## Overview

Wholesalers get squeezed from both sides: cash locked in warehouse inventory and cash locked in customer payment terms (net 30/60/90). Most owners manage this with Excel spreadsheets or gut instinct — both wildly inaccurate.

This app uses AI agents to analyze SKU sales history, current inventory, outstanding invoices, and supplier payment schedules to forecast a 90-day cash flow timeline and recommend whether to liquidate slow stock, chase overdue payments, or secure financing — and in what order.

---

## Features

| Feature | Description |
|---|---|
| Cash Flow Timeline | Daily cash position for the next 90 days with gap alerts |
| Dead Stock List | Slow-moving SKUs ranked by days-of-supply with suggested discount rates |
| Collections Priority | Overdue invoices ranked by aging × amount × late-payment probability |
| Financing Recommendation | How much to borrow, for how long, and how liquidation offsets the gap |
| Audit Compliance Alert | Flags receivables approaching the 90-day write-off threshold |

| Layer | Technology | Notes |

## Database Design

### DynamoDB — Raw Uploads

Table name: `cashflow-raw`

| Key | Type | Description |
|---|---|---|
| `companyId` (PK) | String | Tenant isolation |
| `rowId` (SK) | String | `uploadId#rowIndex` |
| `type` | String | `sales` / `inventory` / `invoice` |
| `data` | Map | Raw CSV row data |
| `uploadedAt` | String | ISO timestamp |

### Aurora PostgreSQL — Structured Analysis

```sql
-- SKU sales history
CREATE TABLE sku_sales_history (
  id          SERIAL PRIMARY KEY,
  company_id  TEXT NOT NULL,
  sku         TEXT NOT NULL,
  sold_qty    INTEGER NOT NULL,
  revenue     NUMERIC(12,2),
  sold_at     DATE NOT NULL
);

-- Current inventory
CREATE TABLE inventory (
  id          SERIAL PRIMARY KEY,
  company_id  TEXT NOT NULL,
  sku         TEXT NOT NULL,
  qty_on_hand INTEGER NOT NULL,
  unit_cost   NUMERIC(12,2),
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);

-- Accounts receivable invoices
CREATE TABLE invoices (
  id           SERIAL PRIMARY KEY,
  company_id   TEXT NOT NULL,
  customer_id  TEXT NOT NULL,
  amount       NUMERIC(12,2) NOT NULL,
  issued_at    DATE NOT NULL,
  due_at       DATE NOT NULL,
  paid_at      DATE,
  status       TEXT DEFAULT 'unpaid'  -- unpaid / partial / paid
);

-- Customer profiles + payment behavior
CREATE TABLE customers (
  id              TEXT PRIMARY KEY,
  company_id      TEXT NOT NULL,
  name            TEXT NOT NULL,
  avg_days_late   NUMERIC(5,1) DEFAULT 0,
  payment_score   NUMERIC(3,1) DEFAULT 5   -- 1-10
);

-- AI forecast results
CREATE TABLE cash_flow_forecast (
  id          SERIAL PRIMARY KEY,
  company_id  TEXT NOT NULL,
  forecast_dt DATE NOT NULL,
  cash_in     NUMERIC(12,2),
  cash_out    NUMERIC(12,2),
  balance     NUMERIC(12,2),
  gap         NUMERIC(12,2),              -- negative = danger
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- Financing scenario comparison
CREATE TABLE loan_scenarios (
  id              SERIAL PRIMARY KEY,
  company_id      TEXT NOT NULL,
  gap_amount      NUMERIC(12,2),
  option_type     TEXT,                   -- bank_loan / liquidate / ar_finance
  amount          NUMERIC(12,2),
  duration_days   INTEGER,
  estimated_cost  NUMERIC(12,2),
  recommended     BOOLEAN DEFAULT FALSE,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

## AWS Setup

1. **DynamoDB** → Create table `cashflow-raw`, PK: `companyId`, SK: `rowId`, on-demand billing
2. **Aurora PostgreSQL Serverless v2** → Create cluster, enable Data API, copy Cluster ARN
3. **Secrets Manager** → Copy the Secret ARN auto-generated by Aurora
4. **IAM User** → Create `cashflow-app`, attach `AmazonDynamoDBFullAccess` + `AmazonRDSDataFullAccess` + `SecretsManagerReadWrite`
5. **RDS Query Editor** → Run `CREATE DATABASE cashflow;`
6. **Local** → Run `npx drizzle-kit push` to auto-create all tables

## Data Flow

```
User uploads CSV
    ↓
api/upload → write raw rows to DynamoDB
    ↓
api/forecast → read from DynamoDB → clean → write to Aurora
    ↓
Claude Agent queries Aurora → generates forecast
    ↓
Dashboard queries Aurora → renders cash flow timeline / dead stock / collections / financing
    ↓
Vercel Cron → refreshes forecast daily
```
